##思路
登录：用户登录后，生成的token是jwt格式，secret是动态生成的
![img.png](img.png)
每次登录的人都会获得一个唯一的动态的secret，这个secret是解析token的信息的

并且当前登录的信息仅仅关联roleIds，每次获取个人权限时动态从缓存中查询最新的。role和menu的信息是单独查询缓存的，这样可以在修改role和menu的关系，只需要单独更新角色和菜单信息，不需要更新每个已登录人的信息，
